# Береза Найф — CRM контекст

## Стек
- Чистый HTML/JS, один файл `index.html` + `bg.jpg`
- Хостинг: **GitHub Pages** → https://bereza-knife.github.io/finance_bereza/
- Данные: **Firebase Realtime Database** (bereza-crm-default-rtdb.europe-west1) — реальный-тайм синк между всеми устройствами. localStorage используется только как кэш для быстрого старта
- Репозиторий: https://github.com/bereza-knife/finance_bereza

## Автодеплой
После каждого Edit/Write файлов срабатывает PostToolUse hook:
`git add . && git commit -m "update" && git push`
Настроен в `.claude/settings.json`

## Финансовая логика

### Шеф-ножи (модели 18–66)
- Миша покупает у Лёши: `себес × 1.5`
- Прибыль Лёши: `себес × 0.5`
- Прибыль Миши: `цена_продажи − себес×1.5`
- Миша видит только цифру «Цена от Лёши», без процентов

### Стейк-ножи (модели 1–17)
- Миша покупает у Лёши: `себес × 1.3`
- Прибыль Лёши: `себес × 0.3 + (цена_продажи − себес×1.3) × 0.5`
- Прибыль Миши: `(цена_продажи − себес×1.3) × 0.5`
- Миша видит только цифру «Цена от Лёши», без процентов

### В аналитике Лёши
- Для заказов Лёши: стандартный `цена − себес`
- Для заказов Миши: формулы выше через `leshaProfit(o)` / `leshaRevenue(o)`

## Интерфейс — вкладки

### Лёша
- **Заказы** — полный CRM: форма нового заказа (с TG-полем), список с фильтрами, кнопка редактирования
- **Производство** — канбан по 8 этапам (Ожидает→Сборка/Финиш), дедлайны, счётчики
- **Клиенты** — карточки клиентов с телефоном, TG, датой последнего заказа, выручкой
- **Аналитика** — выручка/прибыль/маржа с учётом схемы Лёша/Миша

### Миша
- **Заказы** — форма без цен в дропдаунах, показывает «Цена от Лёши» + поля цены продажи/предоплаты/маржи
- **Клиенты** — аналогично Лёше, без финансовых данных

### Григорий
- Заглушка (в разработке)

## Дропдауны — что скрыто
- У Лёши: цены убраны из больстеров, вставок, лайнеров, всечек, форм ручки
- У Миши: цены убраны везде, в дропдаунах только названия

## Редактирование заказов
Кнопка **✎ Редакт.** в каждой карточке (у Лёши и Миши).
- Заполняет форму данными существующего заказа
- Заголовок формы меняется на «Редактирование №X»
- При сохранении — обновляет заказ на месте (статус и дата создания сохраняются)
- Кнопка «✕ Отменить редактирование» сбрасывает форму

## Синхронизация данных
- `orders` + `counter` → Firebase `/orders` и `/counter`
- Производственные данные (этапы, дедлайны) → Firebase `/prod`
- Все три пользователя видят изменения в реальном времени через `onValue`
